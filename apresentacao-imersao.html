O Problema e a Arquitetura
Por que agentes esquecem tudo — e como resolver com um repositório como cérebro permanente
9h15 – 9h35 20 minutos
O que cobrir
  • O problema: agente de IA esquece tudo a cada sessão — sem memória, sem continuidade, sem evolução
  • A solução: repositório GitHub como cérebro permanente — o agente lê de lá toda vez que acorda
  • Diagrama do fluxo: como o agente se conecta ao cérebro
  • Mostrar o repo no GitHub: privado, versionado, organizado por pastas
  • Conceito: o repo é a "mente" da empresa — tudo que o agente sabe está documentado ali
Demos e Arquivos
  • agentes/COMO-CONECTAR.md
    Diagrama do fluxo — como agentes se conectam ao cérebro
  • README.md
    Visão geral do repositório no GitHub
Como fazer
  1. Começar explicando o problema: "Você fala com o ChatGPT hoje, amanhã ele esqueceu tudo"
  2. Mostrar o diagrama em agentes/COMO-CONECTAR.md — fluxo visual
  3. Abrir o repo no GitHub — mostrar que é privado, versionado, com pastas organizadas
  4. Navegar rapidamente pela estrutura de pastas (sem entrar nos arquivos ainda)
  5. Transição: "Agora vamos entrar no cérebro e ver o que tem dentro..."
NÃO mostrar

  • Como instalar o OpenClaw (tutorial já foi enviado)
  • Configuração técnica do git (clone, push, pull)
