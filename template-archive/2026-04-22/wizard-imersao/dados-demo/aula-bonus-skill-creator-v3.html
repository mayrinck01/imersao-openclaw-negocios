O que é

Em 3 frases

😓
O problema

Cada vez que você resolve algo com seu agente, esse conhecimento some quando o chat fecha. Na próxima vez, você explica tudo de novo.

⚡
A solução

O /criar-skill captura o processo que vocês resolveram juntos e ensina o agente de uma vez para sempre.

✅
O resultado

Na próxima vez, ele já sabe. Você não explica de novo. O processo fica documentado no agente para sempre.

📝
É como anotar um processo no manual do seu agente. Só que o agente anota sozinho. Você resolve uma vez, digita /criar-skill, e pronto. O agente nunca mais vai esquecer como fazer.
Conceito

Skill vs Prompt

Entenda a diferença antes de criar.

💬
Prompt
  • Pedido único, para aquele momento
  • Some quando você fecha o chat
  • Você lembra — o agente não
  • Resultado diferente toda vez
⚡
Skill
  • Processo ensinado de vez
  • Fica para sempre no agente
  • Agente faz igual toda vez
  • Resultado consistente, sem explicar
💡
Prompt é o que você pede. Skill é como o agente aprende a fazer. Qualquer processo que você repete mais de 2 vezes é candidato a skill.
O coração da aula

Os dois caminhos para criar uma skill

Dois pontos de partida diferentes. Ambos chegam no mesmo lugar: uma skill funcionando.

Caminho A
Já fiz. Quero preservar.

Você e o agente acabaram de resolver algo juntos — pode ter levado horas, várias tentativas, descoberta de bugs. Foi trabalhoso. Esse conhecimento não pode sumir quando o chat fechar.

Como funciona:
1
Você resolveu o problema Passaram um tempo juntos — testando, ajustando, encontrando o caminho certo. O histórico está ali.
2
Você digita só isso: /criar-skill
3
O agente lê o histórico e cria Extrai o fluxo completo, os workarounds, as armadilhas — tudo que foi descoberto durante a conversa.
4
Você aprova e a skill fica salva Da próxima vez, leva 30 segundos. O agente já sabe o que fazer.
3 exemplos reais:
Exemplo 1 · Nota Fiscal
NF Prefeitura de Curitiba — 4h de tentativa e erro

4 horas testando o portal, encontrando bugs, descobrindo que o campo "Tomador" exige CPF sem máscara, que o ISS retido precisa ser marcado manualmente em dois lugares diferentes, que o portal trava entre 8h–9h pelo volume de acesso. Tudo isso capturado em 2 minutos de /criar-skill. Nunca mais precisou explicar.

✓ Skill criada em 2 min, nunca mais perdeu tempo com o portal
Exemplo 2 · Onboarding
Onboarding de cliente — 12 passos que o agente esquecia no meio

O processo tinha 12 passos: criar conta, configurar integração, mandar email de boas-vindas, agendar call de ativação, adicionar no grupo do WhatsApp, enviar materiais... O agente sempre esquecia algum no meio. Depois do /criar-skill, ele executa os 12 na ordem certa, toda vez, sem pular nenhum.

✓ Zero passos esquecidos desde que virou skill
Exemplo 3 · Relatório
Relatório de métricas — a formatação que nunca saía certa

Tabela específica, ordem específica das métricas, cor vermelha para queda e verde para crescimento, comparação com mês anterior sempre no mesmo formato. Toda vez que pedia o relatório, precisava explicar tudo de novo. Depois do /criar-skill, chegou a palavra "relatório" e o agente já sabia exatamente como montar.

✓ Relatório pronto sem explicar formatação
Prompt exato para digitar: /criar-skill Só isso. O agente lê o histórico da conversa e faz o resto.
Caminho B
Tenho uma ideia. Quero estruturar.

Você tem um processo na cabeça — ou uma ideia de como quer que o agente te ajude — mas não sabe exatamente como estruturar. Não precisa saber. O agente faz perguntas e descobre junto com você.

Como funciona — conversa real:

/criar-skill quero uma skill pra analisar concorrentes

👤
Principalmente preço e o que reclamam nos reviews

👤

⚡

👤
Pesquisa online

Entendido. Vou criar a skill "analise-concorrentes". Você recebe: texto com 3 pontos — preço, principais reclamações nos reviews, e um resumo rápido. Posso confirmar o deploy?

O agente não espera você saber o processo — ele ajuda a descobrir.

Qualidade

O QA e o deploy

Acontece antes de salvar. Você só precisa aprovar.

🛡️
O agente verifica a skill antes de salvar. Ele checa: nome correto, exemplos reais incluídos, casos de erro cobertos, caminho de deploy certo. Se algo falta, ele mesmo corrige. Você só aprova o resultado final.
O que é gerado — 2 arquivos:
skill-name/ ├── SKILL.md # O processo documentado: passos, exemplos, edge cases └── evals/ └── evals.json # Casos de teste — garante que a skill funciona como esperado
📐
evals.json é a prova de que a skill funciona Não é só documentação. São testes reais com input e output esperado. Se a skill for atualizada no futuro, os evals confirmam que continua funcionando certo.
Composição

Skills + Crons + Sub-agentes

Skills sozinhas já valem. Combinadas, viram pipeline.

⏰ Skill + Cron
O agente executa sozinho, no horário certo, sem você lembrar.
❌ Lembrar toda semana de pedir o relatório de métricas
✅ Skill + cron todo domingo às 20h — relatório chega no Telegram sem você fazer nada
🔁
A skill define o que fazer. O cron define quando. Juntos, o agente trabalha no piloto automático.
🤖 Skill + Sub-agente
Manda um assistente fazer uma tarefa com instruções claras.
❌ Spawnar um agente pra "cuidar do conteúdo" — sem saber exatamente o que fazer
✅ Spawnar um agente pra executar a skill carrossel-linkedin com esse input específico
📋
A skill é o briefing do sub-agente. Com ela, o resultado é previsível — igual toda vez.
🔗 Skill + Skill
Um processo que chama outros processos — como um pipeline de conteúdo.
1 vídeo gravado → skill yt-hype gera título e descrição do YouTube
→ skill carrossel-linkedin gera 7 slides do conceito principal
→ skill carrossel-instagram gera versão para o feed
⚡
1 vídeo → 3 skills em sequência → 3 peças de conteúdo prontas. Pipeline completo, sem montar do zero toda vez.
